# Web UI Guide

RustyTalon's web UI is available at `http://localhost:3001` (or your configured host/port) once the agent is running. It provides a full-featured browser interface for chatting, managing memory, monitoring jobs, and configuring the agent.

---

## Logging In

The web UI requires your `GATEWAY_AUTH_TOKEN`. Enter it when prompted, or pass it as a URL parameter:

```
http://localhost:3001?token=your-auth-token
```

The token is stored in your browser session so you only need to enter it once.

---

## Chat Tab

The main interface for talking to the agent.

### Sending Messages

Type in the input box at the bottom and press **Enter** (or click Send). Responses stream in real time as the agent thinks and replies.

### Slash Commands

Type these directly in the chat input:

| Command | What it does |
|---------|-------------|
| `/help` or `/?` | Show available commands |
| `/tools` | List all tools the agent can use |
| `/version` | Show the agent version |
| `/model` | Show the current LLM model |
| `/model <name>` | Switch to a different model (e.g. `/model gpt-4o`) |
| `/ping` | Check agent responsiveness |
| `/debug` | Show debug information |
| `/undo` | Undo the last conversation turn |
| `/redo` | Redo a previously undone turn |
| `/compact` | Compress conversation history to free context window space |
| `/clear` | Clear the current thread and start fresh |
| `/summarize` or `/summary` | Summarize the current conversation |
| `/suggest` | Ask the agent to suggest next steps |
| `/heartbeat` | Trigger a manual heartbeat check |
| `/new` or `/thread new` | Start a new conversation thread |
| `/thread <uuid>` | Switch to a different conversation thread |
| `/stop` or `/interrupt` | Stop the agent mid-response |
| `/quit` or `/exit` | Shut down the agent |

### Tool Approval

When the agent wants to use a potentially sensitive tool (shell commands, file writes, HTTP requests), it will pause and ask for your approval. You can:

- Type `yes` or `y` to approve once
- Type `always` to approve this tool for the rest of the session
- Type `no` or `n` to deny

### Multiple Threads

Each conversation thread is isolated. Use `/new` to start a fresh context, or `/thread <uuid>` to switch between existing threads. The **Threads** list in the sidebar shows all available threads.

## Memory Tab

Browse and manage the agent's persistent memory — a flexible filesystem of documents the agent reads and writes to remember things about you and your projects.

### Directory Tree

The left panel shows the workspace filesystem. Common paths:

| Path | Purpose |
|------|---------|
| `MEMORY.md` | Long-term curated memory |
| `AGENTS.md` | Behavior instructions for the agent |
| `USER.md` | Information about you |
| `SOUL.md` | Agent's core values |
| `IDENTITY.md` | Agent's name and personality |
| `HEARTBEAT.md` | Checklist the agent runs periodically |
| `daily/YYYY-MM-DD.md` | Daily logs |
| `context/` | Identity and priority documents |
| `projects/` | Project-specific notes |

### Reading Files

Click any file in the tree to view its contents in the right panel.

### Writing Files

Use the write interface to create or update any file. You can also ask the agent in chat to update memory for you: *"Remember that I prefer dark mode"* or *"Write a note about project Alpha to projects/alpha/notes.md"*.

### Searching Memory

The search bar performs a hybrid search combining keyword matching and semantic similarity. Useful for finding notes when you don't remember the exact path: *"when did I last work on the API?"*.

## Jobs Tab

Monitors background jobs — tasks the agent is executing in parallel or inside Docker sandbox containers.

### Job States

| State | Meaning |
|-------|---------|
| **Pending** | Waiting to start |
| **In Progress** | Currently executing |
| **Completed** | Finished successfully |
| **Failed** | Encountered an error |
| **Stuck** | Detected as hung; will attempt self-repair |

### Job Actions

- **Cancel** — Stop a running job
- **Restart** — Retry a failed job
- **View Events** — See the full event log for a job
- **Browse Files** — Read files generated by the job (e.g. code the agent wrote)

### Sandbox Jobs

When Docker sandbox mode is enabled (`SANDBOX_ENABLED=true`), long-running or untrusted tasks execute in isolated containers. You can send follow-up prompts to Claude Code sandbox jobs from this tab.

## Routines Tab

Create and manage automated tasks that run on a schedule or in response to events.

### Routine Types

| Type | Trigger |
|------|---------|
| **Cron** | Runs on a time schedule (e.g. every day at 9am) |
| **Event** | Triggered by an internal agent event |
| **Webhook** | Triggered by an incoming HTTP request |
| **Manual** | Only runs when you click "Trigger" |

### Creating a Routine

1. Click **New Routine**
2. Set the name and description
3. Choose a trigger type and configure it
4. Write the prompt the agent will receive when the routine fires
5. Optionally set guardrails (max run time, allowed tools)
6. Click **Save**

### Routine Controls

- **Toggle** — Enable or disable without deleting
- **Trigger** — Run the routine immediately (works for any type)
- **History** — See the last 50 executions with status and output

## Extensions Tab

Install, authenticate, and manage tools and integrations beyond the built-in set.

### Extension Types

| Type | Description |
|------|-------------|
| **WASM** | Sandboxed tools built in Rust, run in the agent process |
| **MCP** | Model Context Protocol servers (external process, any language) |

### Installing an Extension

1. Click **Install Extension**
2. Choose the type (WASM or MCP)
3. Provide the path (WASM binary) or URL (MCP server)
4. Click **Install**

### Authenticating a Tool

Some tools need credentials (API keys, OAuth tokens). After installing:

1. Select the extension
2. Click **Authenticate**
3. Follow the prompts — the agent handles OAuth flows or manual token entry

### Activating / Deactivating

Installed tools can be activated (available to the agent) or deactivated (installed but not used). Toggle activation without losing configuration.

## Logs Tab

A real-time stream of the agent's internal log output. Useful for:

- Watching what the agent is doing
- Debugging errors
- Seeing tool execution details

Logs replay recent history when you connect so you don't miss events that happened before you opened the tab.

---

## Settings Tab

Configure per-user settings stored in the database. These persist across sessions.

Common settings:

| Key | Description |
|-----|-------------|
| `theme` | UI theme (`dark` or `light`) |
| `default_model` | Preferred LLM model for this user |

You can also export and import settings as JSON for backup or transfer between instances.

---

## Provider Health

Check the status of your configured LLM providers from the API (no dedicated UI tab, but accessible via the API):

```bash
curl http://localhost:3001/api/providers/health \
  -H "Authorization: Bearer $GATEWAY_AUTH_TOKEN"
```

```bash
curl http://localhost:3001/api/providers/costs \
  -H "Authorization: Bearer $GATEWAY_AUTH_TOKEN"
```

These show per-provider availability scores and total usage costs.

## Real-Time Streaming

The web UI uses Server-Sent Events (SSE) to receive live updates. If you prefer WebSockets, connect to `/api/chat/ws`. Both deliver the same event types:

| Event | Meaning |
|-------|---------|
| `thinking` | Agent is reasoning/planning |
| `message` | Text response from the agent |
| `tool_call` | Agent is calling a tool |
| `tool_result` | Tool returned a result |
| `approval_request` | Agent needs your approval to proceed |
| `auth_request` | A tool needs credentials |
| `job_update` | A sandbox job changed state |
| `error` | Something went wrong |
